# 회전형 20구역 왁스 셸 디자인

## 목표

왁뿌볼을 드래그해 돌려가며 서로 다른 표면 20곳을 깨야 말랑이가 나타나는 상호작용으로 변경한다. 파괴는 누른 위치에서 시작하고, 왁스 균열과 파편은 불규칙하며 중력 방향으로 낙하한다. 말랑이 효과음은 물 흐르는 인상을 없앤 짧고 쫀득한 `찹·뽁` ASMR로 교체한다.

## 사용자 흐름

1. 새 게임은 무작위 왁스·말랑이 색상 조합과 손상되지 않은 구형 왁스 셸로 시작한다.
2. 사용자가 볼을 누른 채 움직이면 볼이 가로·세로로 회전한다.
3. 이동량이 작은 짧은 터치는 화면에서 누른 구면 위치를 파괴한다.
4. 표면은 20개의 고정된 구면 영역으로 구성된다. 처음 파괴한 영역만 진행도에 포함한다.
5. 이미 깨진 영역을 다시 누르면 추가 진행이나 중복 파편을 만들지 않는다. 짧은 비활성 촉각 애니메이션만 허용하며 파괴음은 재생하지 않는다.
6. 깨진 균열과 구멍은 볼을 회전할 때 해당 구면 위치를 유지한 채 함께 이동한다.
7. 20개 영역을 모두 파괴하면 남은 왁스가 아래로 무너지고 셸이 완전히 사라진 뒤 말랑이만 남는다.
8. 말랑이는 제한 없이 계속 누를 수 있다. 5회부터 기존과 같이 `다시하기` 버튼을 표시한다.

## 상호작용 모델

- 포인터 입력은 `pointerdown`, `pointermove`, `pointerup`, `pointercancel`로 통합한다.
- 누른 뒤 총 이동 거리가 8 CSS 픽셀 미만이면 터치, 이상이면 회전 드래그로 판정한다.
- 드래그는 수평 이동을 요(yaw), 수직 이동을 피치(pitch)에 반영한다. 피치는 앞뒤가 뒤집히지 않도록 제한하고 관성 회전은 넣지 않는다.
- 화면의 터치 위치를 단위 구의 전면 교차점으로 환산한 뒤 현재 회전의 역행렬을 적용해 볼의 로컬 좌표를 얻는다.
- 20개 영역 중심은 피보나치 구면 분포로 균등 배치한다. 로컬 터치 벡터와 내적이 가장 큰 영역을 선택한다.
- 각 영역은 `intact` 또는 `broken` 상태 하나만 가진다. 전체 진행도는 `broken` 영역 개수이며 최대값은 정확히 20이다.

## 렌더링

기존 고정 SVG 파괴 레이어를 볼 크기에 맞춘 고해상도 Canvas 셸 렌더러로 교체한다. 말랑이 본체와 페이지 레이아웃은 기존 DOM/CSS를 유지한다.

Canvas는 매 프레임 다음 순서로 그린다.

1. 구형 왁스 바탕: 색상 테마, 비대칭 명암, 미세한 기공과 얇은 왁스 결.
2. 깨지지 않은 영역 경계의 미세한 응력선: 진행도가 높아질수록 약하게 증가하지만 화살표나 규칙적인 방사형 패턴은 사용하지 않는다.
3. 깨진 영역: 결정적 시드로 생성한 7~11개 꼭짓점의 오목·볼록 혼합 다각형, 밝은 파단면, 어두운 깊이, 2~4개의 짧은 가지 균열.
4. 전면 하이라이트와 가장자리 음영으로 구체감을 복원한다.
5. 활성 파편: 원래 깨진 영역과 같은 색·결을 가진 작은 불규칙 조각 2~4개.

구면 뒤쪽에 있는 손상 영역은 그리지 않는다. 앞쪽으로 회전하면 다시 동일한 위치와 형태로 나타난다. 파괴 형태는 영역별 고정 시드로 생성해 회전 중 모양이 흔들리거나 바뀌지 않게 한다.

## 파편 물리

- 새 파편의 초기 수직 속도는 0 이상으로 설정해 위로 솟지 않게 한다.
- 매 프레임 `y = y0 + vy*t + 0.5*g*t²`를 적용하고, 수평으로는 작은 일정 속도와 회전만 준다.
- 파편은 화면 아래쪽을 벗어나거나 900ms가 지나면 제거한다.
- 20번째 영역 파괴 시 셸의 잔여 부분은 180ms 동안 균열이 번진 뒤 여러 조각으로 분리되어 모두 아래로 낙하한다. 전환 중 말랑이는 아래에서 서서히 드러난다.

## 사운드

- 기존 왁스 파괴 샘플과 음량 기준은 유지한다.
- 말랑이 샘플은 짧은 접착성 슬라임 소리만 사용한다. 물 흐름, 연속 기포, 긴 잔향이 들리는 구간은 제외한다.
- 샘플 시작과 끝을 짧게 페이드하고 180Hz 이하의 불필요한 저역을 줄인다.
- 재생 길이는 약 120~320ms로 제한하고, 체감 음량은 왁스 파괴 사운드와 비슷하게 정규화한다.
- 3개 이상의 변형 샘플을 무작위 재생하며 재생 속도를 0.97~1.04 범위에서만 변화시킨다.

## 상태와 표시

- 왁스 단계 최대값은 `20`이다.
- 화면 상태 문구와 접근성 라이브 영역은 `깨진 표면 n / 20`을 알린다.
- 같은 영역을 다시 누르면 진행도는 변하지 않고 `이미 깨진 곳이에요. 볼을 돌려보세요`를 안내한다.
- 드래그 중에는 파괴 입력이나 소리를 발생시키지 않는다.
- 셸이 완전히 사라지기 전에는 말랑이 입력을 받지 않는다.

## 코드 경계

- `public/game/surface.js`: 구면 영역 생성, 좌표 변환, 영역 선택, 손상 데이터와 결정적 파괴 형태.
- `public/game/wax-renderer.js`: Canvas 크기 조정, 왁스 표면·균열·구멍·파편 렌더링과 낙하 애니메이션.
- `public/game/app.js`: 게임 상태, 포인터 제스처 구분, 단계 전환, UI와 오디오 연결.
- `public/game/audio.js`: 샘플 목록, 음량·재생 속도, 짧은 보조 합성음.
- `public/game/index.html`: Canvas 셸 요소와 기존 말랑이 요소.
- `public/game/styles.css`: 회전 입력, Canvas 레이어, 셸 붕괴·말랑이 전환 스타일.

## 테스트

- 20개 구면 영역이 생성되고 모두 단위 구 위에 분포하는지 검증한다.
- 동일한 로컬 좌표를 반복 터치해도 진행도가 한 번만 증가하는지 검증한다.
- 서로 다른 19개 영역에서는 왁스 단계가 유지되고 20번째에서만 말랑이 단계로 전환되는지 검증한다.
- 화면 좌표와 회전값을 바꿨을 때 올바른 로컬 영역이 선택되는지 검증한다.
- 드래그 임계값 이상에서는 파괴가 발생하지 않는지 검증한다.
- 파편의 초기 수직 속도와 이후 위치가 항상 중력 방향으로 증가하는지 검증한다.
- 말랑이 오디오 목록과 믹스 범위, 공개 페이지의 20단계 안내를 검증한다.
- 전체 프로젝트 빌드와 기존 말랑이 반복·다시하기·무작위 디자인 테스트를 통과시킨다.

## 완료 기준

- 사용자는 볼을 실제로 드래그해 뒤쪽 표면까지 돌려볼 수 있다.
- 서로 다른 20개 영역을 깨야만 말랑이가 나타난다.
- 누른 위치에 불규칙한 왁스 파단면이 생기며 그 자국은 회전해도 볼에 고정된다.
- 파편은 위로 튀지 않고 아래로 낙하한다.
- 왁스가 완전히 사라진 후에는 말랑이만 보인다.
- 말랑이 소리에서 시냇물 같은 연속 수류감이 들리지 않고, 짧고 부드러운 `찹·뽁` 질감이 난다.
- 모바일에서 터치·드래그가 충돌하지 않고 60fps에 가깝게 동작한다.
